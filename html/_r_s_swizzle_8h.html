Swizzles the class method of the class with the new implementation.

Example for swizzling +(int)calculate:(int)number; method:

1 RSSwizzleClassMethod(classToSwizzle,
2  @selector(calculate:),
3  RSSWReturnType(int),
4  RSSWArguments(int number),
5  RSSWReplacement(
6 {
7  // Calling original implementation.
8  int res = RSSWCallOriginal(number);
9  // Returning modified return value.
10  return res + 1;
11 }));

Swizzling is fully thread-safe.

Parameters
classToSwizzleThe class with the method that should be swizzled.
selectorSelector of the method that should be swizzled.
RSSWReturnTypeThe return type of the swizzled method wrapped in the RSSWReturnType macro.
RSSWArgumentsThe arguments of the swizzled method wrapped in the RSSWArguments macro.
RSSWReplacementThe code of the new implementation of the swizzled method wrapped in the RSSWReplacement macro.

Swizzles the instance method of the class with the new implementation.

Example for swizzling -(int)calculate:(int)number; method:

1 RSSwizzleInstanceMethod(classToSwizzle,
2  @selector(calculate:),
3  RSSWReturnType(int),
4  RSSWArguments(int number),
5  RSSWReplacement(
6 {
7  // Calling original implementation.
8  int res = RSSWCallOriginal(number);
9  // Returning modified return value.
10  return res + 1;
11 }), 0, NULL);

Swizzling frequently goes along with checking whether this particular class (or one of its superclasses) has been already swizzled. Here the RSSwizzleMode and key parameters can help. See +[RSSwizzle swizzleInstanceMethod:inClass:newImpFactory:mode:key:] for details.

Swizzling is fully thread-safe.

Parameters
classToSwizzleThe class with the method that should be swizzled.
selectorSelector of the method that should be swizzled.
RSSWReturnTypeThe return type of the swizzled method wrapped in the RSSWReturnType macro.
RSSWArgumentsThe arguments of the swizzled method wrapped in the RSSWArguments macro.
RSSWReplacementThe code of the new implementation of the swizzled method wrapped in the RSSWReplacement macro.
RSSwizzleModeThe mode is used in combination with the key to indicate whether the swizzling should be done for the given class. You can pass 0 for RSSwizzleModeAlways.
keyThe key is used in combination with the mode to indicate whether the swizzling should be done for the given class. May be NULL if the mode is RSSwizzleModeAlways.
Returns
YES if successfully swizzled and NO if swizzling has been already done for given key and class (or one of superclasses, depends on the mode).

typedef id(^ RSSwizzleImpFactoryBlock) (RSSwizzleInfo *swizzleInfo)

A factory block returning the block for the new implementation of the swizzled method.

You must always obtain original implementation with swizzleInfo and call it from the new implementation.

Parameters
swizzleInfoAn info used to get and call the original implementation of the swizzled method.
Returns
A block that implements a method. Its signature should be: method_return_type ^(id self, method_args...). The selector is not available as a parameter to this block.

typedef NS_ENUM ( NSUInteger  ,
RSSwizzleMode   
)

RSSwizzle always does swizzling.

RSSwizzle does not do swizzling if the same class has been swizzled earlier with the same key.

RSSwizzle does not do swizzling if the same class or one of its superclasses have been swizzled earlier with the same key.

Note
There is no guarantee that your implementation will be called only once per method call. If the order of swizzling is: first inherited class, second superclass, then both swizzlings will be done and the new implementation will be called twice.
